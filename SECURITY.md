## 报告安全漏洞

如果您发现了安全漏洞，请 **不要** 创建公开的 Issue。

### 报告流程

1. **发送邮件至维护者**
   - 邮箱：请通过 GitHub 个人资料获取联系方式
   - 主题：`[SECURITY] A-Termux-Integration-Toolbox - 漏洞描述`

2. **邮件内容应包含**
   - 漏洞的详细描述
   - 复现步骤
   - 受影响的版本
   - 可能的修复建议（如有）
   - 您的联系方式（可选）

3. **等待回复**
   - 维护者会在 48 小时内确认收到报告
   - 会在 7 天内提供初步评估
   - 修复后会通知您并公开致谢（如同意）

### 安全更新流程

1. 收到报告后，维护者会验证漏洞
2. 在私密环境中开发修复
3. 发布安全更新版本
4. 公开披露漏洞详情（在修复后）

## 安全最佳实践

### 对于用户

1. **保持更新**
   - 始终使用最新版本
   - 关注安全公告

2. **权限管理**
   - 仅授予必要的权限
   - 谨慎使用 Root 功能

3. **脚本安全**
   - 仔细审查导入的自定义脚本
   - 仅从可信来源下载远程脚本
   - 定期检查脚本内容

4. **数据保护**
   - 定期备份配置
   - 不在脚本中硬编码敏感信息

### 对于贡献者

1. **代码审查**
   - 所有代码更改都需要审查
   - 特别关注输入验证和命令执行

2. **依赖管理**
   - 及时更新依赖项
   - 检查依赖项的安全漏洞

3. **测试**
   - 添加安全相关的测试用例
   - 测试边界情况和错误处理

## 安全相关配置

### 禁用远程脚本功能（如需更高安全性）

```bash
# 编辑配置文件
rm ~/.config/ATT/remote_sources.conf
# 创建空文件防止重新生成
touch ~/.config/ATT/remote_sources.conf
chmod 444 ~/.config/ATT/remote_sources.conf
```

### 限制脚本执行权限

```bash
# 仅允许特定用户访问脚本目录
chmod 700 ~/.ATT/custom_scripts
```
